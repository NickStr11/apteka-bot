# AI Agent Rules

## Память
- **Читай** `DEV_CONTEXT.md` при открытии нового чата
- **Обновляй** `DEV_CONTEXT.md` после изменений
- При длинном контексте (скроллбар <10%) → предложи новый чат

## Работа с файлами
- Проверяй существование перед изменением (`list_dir`, `view_file`)
- Используй абсолютные пути
- Не создавай фантомные файлы

## Коммуникация
- Минимум "воды", максимум конкретики
- Markdown для структуры
- Не хвали без дела

## Обновление DEV_CONTEXT.md
**Когда:** создание файлов, реализация функций, баги, решения, завершение этапа

**Формат:**
```markdown
### YYYY-MM-DD HH:MM — Название
- ✅ Что сделано
- Решения: [обоснование]
- Следующий шаг: [задача]
```

## При длинном контексте
> "Контекст большой. Рекомендую:
> 1. Новый чат
> 2. Написать: 'Продолжи [PROJECT]'
> Прочитаю DEV_CONTEXT.md и продолжу."

## Специфика проекта
> Замени на свой стек (Python/JS/Go и т.д.)
